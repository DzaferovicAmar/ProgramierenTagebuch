1. General Requirements

  1. Display a white canvas to the user

  2. When the user clicks on the canvas:

    1. If the space is empty a shape is rendered at this location — with the click position becoming the center point of the new shape

    2. If a shape is already present at this location it gets removed

  3. New shapes are created in the following, circular order:

    1. Square

    2. Circle

    3. Rectangle

    4. Triangle

  4. For new shapes color and size are chosen randomly

    1. Size has to be constrained to the boundaries of the canvas (= don’t draw outside)

    2. The triangle is equilateral

    3. For the rectangle both width and height are chosen randomly which can lead to a larger height than width ⇒ the rectangle is rendered in vertical orientation

  5. New shapes get assigned a unique, incrementing ID

  6. The final application should work like in the following sample run

2. Tasks

Every shape has:

  • Several common properties:

    1. An ID

    2. A center point (x,y)

    3. A color

  • Two common functionalities:

    1. It can draw itself on the canvas

    2. It can tell you if a point (x,y) is contained within the area of the shape

The provided parts of Program & ShapeGenerator assume that there is a class Shape ⇒ if you use different names make sure to change the provided code accordingly

2.2.2. Functionality

  • You have to use inheritance

  • Make sure to avoid code duplication wherever possible in a sensible way

  • Use the same color for both the borderline and the fill content

  • Sizes are generated in the following boundaries:

    • Rectangle: width or height 10-150

    • Triangle: side length 10-150

    • Circle: radius 5-100

    • Square: side length 10-120

  • Size may have to be reduced if the user clicked close to the border of the canvas, we don’t want to draw outside the canvas

  • It is necessary to know if a point lies within the area of a shape, because we have to know if and which shape the user clicked (or if the click hit empty canvas)

  • If two shapes overlap the click 'hits' the one 'on top' ⇒ the one added later

  • Actions (adding or removing a shape) are logged to the console

3. Implementation Hints

Here are a couple hints that may help you implement the required functionality.

  • You can use these colors:

private static readonly IBrush[] colors =
    {
        Brushes.Blue,
        Brushes.Cyan,
        Brushes.DarkGreen,
        Brushes.Firebrick,
        Brushes.Lime,
        Brushes.Orange,
        Brushes.Plum,
        Brushes.Yellow
    };

  • The following function can be used to determine if a point lies on the right hand side of an edge (be careful with clockwise or counter-clockwise edge iteration):

private static bool IsOnRightSideOfEdge(Point edgeStart, Point edgeEnd, Point pointToCheck)
{
    double d = (edgeEnd.X - edgeStart.X) * (pointToCheck.Y - edgeStart.Y)
               - (pointToCheck.X - edgeStart.X) * (edgeEnd.Y - edgeStart.Y);

    return d > 0D;
}

  • A (filled) triangle cannot be drawn by using three lines, because then you won’t be able to fill it. Instead, you may use the SimpleDrawing.DrawPolygonByPath method.
